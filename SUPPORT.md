# Support

Welcome to the FreClean Support Guide.

This document explains how to get help, report problems, ask questions, and contact the FreClean team.

---

# Getting Help

Before opening a new issue, please:

- Read the README.md.
- Review the documentation in the `docs/` directory.
- Check the Frequently Asked Questions (FAQ).
- Search existing GitHub Issues.
- Search GitHub Discussions.

Many common questions have already been answered.

---

# Support Channels

## GitHub Discussions

Use GitHub Discussions for:

- General questions
- Community conversations
- Product ideas
- Business discussions
- Best practices
- Feedback

---

## GitHub Issues

Open an Issue only when reporting:

- Software bugs
- Documentation errors
- Broken links
- Packaging or branding mistakes
- Reproducible technical problems

When creating an issue, include:

- A clear title
- A detailed description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Environment information

---

## Security Issues

**Do not report security vulnerabilities publicly.**

Please follow the instructions in `SECURITY.md`.

# Product Support

If your question concerns a FreClean product, please provide:

- Product name
- Product version (if applicable)
- Purchase location
- Date of purchase
- Batch or lot number (if available)
- Description of the issue
- Photos (if relevant)

This information helps us investigate more efficiently.

---

# Service Support

For cleaning service requests or support, include:

- Type of service
- Location
- Preferred date
- Description of the request
- Photos (if applicable)

---

# Documentation Support

Documentation improvements are always welcome.

If you notice:

- Incorrect information
- Missing sections
- Broken links
- Formatting problems
- Translation issues

Please open a Documentation Issue or submit a Pull Request.

---

# Response Times

Target response times are:

| Request Type | Target Response |
|--------------|----------------:|
| Critical Security | Within 24 hours |
| Service Disruption | 1–2 business days |
| Technical Bug | 2–5 business days |
| Documentation | 3–7 business days |
| Feature Request | Reviewed during roadmap planning |
| General Question | 2–5 business days |
| Business Inquiry | 3–7 business days |
| Partnership Request | 5–10 business days |

These are targets and may vary depending on workload.
