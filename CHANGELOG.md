## Versioning Policy

FreClean follows Semantic Versioning.

Version numbers are formatted as:

```text
MAJOR.MINOR.PATCH
```

### MAJOR

Incompatible or significant changes.

Examples:

- Major platform redesign.
- Breaking architectural changes.
- New business direction.

### MINOR

New functionality while maintaining compatibility.

Examples:

- New services.
- New products.
- New documentation.
- Additional features.

### PATCH

Bug fixes and documentation improvements.

Examples:

- Documentation corrections.
- Security fixes.
- Performance improvements.
- Typographical corrections.

---

## Release Types

### Stable Release

Production-ready version.

Example:

```text
v1.0.0
```

### Minor Release

Additional features.

Example:

```text
v1.1.0
```

### Patch Release

Maintenance update.

Example:

```text
v1.1.1
```

### Major Release

Significant changes.

Example:

```text
v2.0.0
```

## Release Checklist

Before publishing a new release:

- Documentation updated.
- Version number updated.
- CHANGELOG updated.
- Security review completed.
- Quality review completed.
- Tests completed.
- GitHub Release published.
